Most used git commands

git clone Clone a repository into a new directory
git add Add file contents to the index
git commit Record changes to the repository
git reset reset commited file
git restore --staged * undo from (git add) command
git clean -n check removing uncommited new files
git clean -f removing uncommited new files
git push Push the changes to the remote repository
git push -u origin master Pull and Push the changes to the remote repository
git pull Fetch and merge changes from the remote repository
git status Show the working tree status
git branch list of branches
git branch master create master
git branch -m master rename current to master
git branch -d master to delete master
git branch -D master force delete master
git checkout Switch branches or restore working tree files
git checkout -b master create and Switch branche to master
git merge Join two or more development histories together
git merge dev merge current branch with dev
git stash need git add first , move current edits to temp stash
git stash save "message" need git add first , move current edits to temp stash with custom message
git stash list get stash list
git stash pop remove edits from stash to current branch
git stash pop stash@{0} remove edits with the index 0 from stash
git stash apply send edits from stash to current branch without removing from stash
git stash drop destroy last stack element edits in stash
git stash drop stash@{0} destroy edits with index 0 in stash
git stash show show info about stash
git stash show stash{0} show info about stash with index 0
git stash clear clear stash
git log Show commit logs
git config --global alies.cm "commit -m" when run (git cm) will run git commit -m
git config --global alies.cm will return cm shortcut value
git config --global edit will open config file

How to generate public key and authenticate with GitHub

To generate a public key, you need to use the ssh-keygen command. This will create a pair of files in your ~/.ssh directory: id_rsa (private key) and id_rsa.pub (public key).

To authenticate with GitHub, you need to copy the contents of your id_rsa.pub file and paste it into your GitHub account settings under SSH and GPG keys. Then, you can use the ssh -T git@github.com command to test your connection.

For more details, you can visit the official GitHub documentation: https://docs.github.com/en/authentication/connecting-to-github-with-ssh
